# 💍 Selpic — 웨딩 사진 셀렉 웹앱

신랑과 신부가 **각자 자기 기기에서** 동일한 웨딩 사진(1000장 이상, jpg)을 넘겨보며
1~5점으로 매기고, **두 사람 점수 합계가 높은 사진의 파일명만** 최종으로 수합하는 도구입니다.

- 🔒 **사진은 업로드되지 않습니다.** 브라우저가 로컬 폴더의 사진을 그 자리에서 읽어 보여줄 뿐,
  네트워크로 오가는 것은 `파일명 + 점수`(수십 KB)뿐입니다.
- ☁️ 취합은 무거운 DB 없이 **구글 시트 + Apps Script**로. 시트 자체가 결과표이자 백업입니다.
- 🖥 **크롬/엣지 데스크톱** 기준. (파이어폭스/사파리는 폴더 선택 폴백으로 보기·평점만 가능)

---

## 1. 백엔드(구글 시트) 준비 — 한 명만 1회

1. 새 구글 시트를 만듭니다.
2. **확장 프로그램 → Apps Script**를 열고, `apps-script/Code.gs` 내용을 전부 붙여넣고 저장합니다.
3. 우측 상단 **배포 → 새 배포 → 유형: 웹 앱**
   - 실행 계정: **나**
   - 액세스 권한: **모든 사용자(Anyone)**
4. 발급된 **웹 앱 URL**(`https://script.google.com/macros/s/.../exec`)을 복사합니다.
   → 이 URL을 신랑·신부 두 사람이 앱 접속 화면에 똑같이 입력합니다.

> `ratings` 탭은 자동 생성됩니다. 시트를 열어두면 점수가 실시간으로 쌓이는 걸 볼 수 있어요.

## 2. 앱 실행

정적 파일이라 빌드가 필요 없습니다. 프로젝트 폴더에서:

```bash
npx serve .
```

출력된 주소(예: `http://localhost:3000`)를 **크롬/엣지**로 엽니다.
여러 기기에서 쓰려면 [Netlify Drop](https://app.netlify.com/drop)이나 GitHub Pages 등
무료 정적 호스팅에 폴더째 올리고 URL을 공유하면 됩니다.

> `file://`로 직접 열면 폴더 접근 API가 제한될 수 있으니 `npx serve` 또는 호스팅을 권장합니다.

## 3. 사용법

1. 접속 화면에서 **역할(신랑/신부)** 선택 → **사진 폴더 선택**.
   (웹앱 URL은 `src/config.js`의 `DEFAULT_SHEET_URL`에 미리 넣어두면 입력칸이 숨겨져 매번 입력할 필요가 없습니다.
   비워두면 접속 화면에서 직접 입력. ⚠️ public 저장소면 URL이 노출되니, 사용 후 Apps Script 재배포로 무효화 권장.)
2. **🖼 1장** 모드: 방향키 `←/→`로 넘기고 숫자키 `1~5`로 즉시 평점(`0`은 취소). 별을 눌러도 됩니다.
   - **모바일**: 사진을 좌우로 **스와이프**해서 다음/이전(확대 안 된 상태). 이전/다음 버튼은 하단 양쪽 끝,
     별점은 가운데 크게 배치(엄지 조작).
   - **확대**: 마우스 휠/트랙패드로 확대·축소(커서 기준), 확대 상태에서 **드래그로 이동**,
     **더블클릭**으로 맞춤↔2.5배 토글. 오른쪽 아래 줌 바(`− + ⤢`)와 키보드 `+`/`−`/`f`도 사용.
     **모바일**: 두 손가락 **핀치**로 확대, 한 손가락 드래그로 이동, 더블탭 토글.
     사진을 넘기면 확대는 자동으로 화면맞춤으로 초기화됩니다.
3. **▦ 그리드**: 전체를 한눈에. 썸네일의 `⚖` 버튼으로 비교에 담고, 클릭하면 1장 모드로 이동.
4. **⚖ 비교**: 담은 2~4장을 나란히 보며 평점.
5. **필터**: `합계≥`, `내가 매긴 것/안 매긴 것`, `합계순` 정렬.
6. **사진 추가**: **그리드 끝의 `＋ 사진 추가` 타일**을 눌러 목록에 사진을 더 **덧붙이기**
   (데스크톱=폴더, 아이폰=여러 장 선택). 평점은 파일명 기준이라 기존 점수는 유지됩니다.

## 4. 최종 파일명 수합 (가장 중요)

원하는 필터를 걸어둔 상태(예: `합계≥ 8`, `합계순`)에서 **⬇ 최종 내보내기**를 누르면:

- `selpic-YYYY-MM-DD.txt` — 파일명 한 줄에 하나
- `selpic-YYYY-MM-DD.csv` — `filename, groom, bride, total`

또는 **구글 시트에서 직접** `total` 열로 정렬·필터한 뒤 `filename` 열을 복사해도 됩니다.

## 검증(테스트)

브라우저 없이 핵심 로직과 동기화 계약을 검증하는 하니스가 있습니다
(점수 병합·역할별 합계·필터/정렬·내보내기 + mock 시트로 신랑↔신부 동기화):

```bash
npm test        # 로직·동기화 계약 (node, 빠름)
npm run test:ui # 실제 크롬(Playwright)으로 UI end-to-end + 스크린샷
```

`test:ui`는 최초 1회 브라우저 설치가 필요합니다: `npm i && npx playwright install chromium`.
UI 테스트는 앱을 정적 서버로 띄우고 mock 시트 백엔드에 연결해, 폴더 로드→평점(키보드)→
그리드→내보내기→신랑↔신부 동기화까지 실제 브라우저에서 확인하고 `test/screenshot.png`를 남깁니다.

실제 배포한 시트 URL로 브라우저 CORS 왕복(POST/GET)까지 확인하려면:

```bash
SELPIC_URL='https://script.google.com/macros/s/.../exec' node test/real-verify.mjs
```

> 이 테스트는 시트 `ratings` 탭에 `test_selpic.jpg`, `sttest_*` 같은 **테스트 행을 남깁니다.**
> 실제 사용 전에 해당 행들을 지우세요.

## 파일 구조

```
index.html            앱 셸(접속 화면 + 앱 화면)
styles.css
src/
  app.js              화면 전환/툴바 배선/뷰 라우팅
  ratings.js          상태 저장소 + 점수 병합/필터/정렬
  fs.js               로컬 폴더 접근(File System Access + 폴백), 이미지 로드
  sync.js             동기화 오케스트레이터(백엔드 어댑터 위에서 pull/push 타이밍)
  backends/
    sheets-backend.js 구글 시트 어댑터 (배포 시 supabase-backend.js로 교체)
  ui-rate.js          1장 모드(+키보드)
  ui-grid.js          썸네일 그리드(지연 로딩)
  ui-compare.js       비교(2~4분할) 뷰
  export.js           txt/csv 내보내기
  ui-common.js        DOM 헬퍼 + 별점 위젯
apps-script/Code.gs   구글 시트에 붙이는 백엔드
MIGRATION.md          배포 확장(구글 시트 → Supabase) 가이드
```

> **배포 확장**: 동기화는 교체 가능한 어댑터로 분리돼 있어, 다른 부부들에게 배포할 땐
> 클라이언트를 건드리지 않고 백엔드 어댑터만 교체하면 됩니다. → `MIGRATION.md`

## 참고 / 주의

- 두 기기의 **사진 파일명이 동일**해야 점수가 매칭됩니다(같은 원본 세트면 자동 충족).
- **아이폰**: 폴더 선택(File System Access)이 iOS에선 불가 → 자동으로 **다중 파일 선택**으로 전환됩니다.
  파일 앱에서 웨딩 사진(jpg)을 전체 선택하세요. 단, iOS는 폴더를 기억 못 해 **접속할 때마다 다시 선택**해야 하고,
  사진이 아주 많으면 폰 메모리 부담이 있습니다(주 작업은 데스크톱 권장). 원본 jpg를 골라야 파일명이 맞습니다.
- 동기화는 즉시 푸시가 아니라 **몇 초 간격 폴링**(near-real-time)입니다. 각자 매기다 합치는 용도엔 충분해요.
- Apps Script 웹앱을 "모든 사용자"로 열기 때문에 **URL이 사실상 비밀키**입니다. 외부에 노출하지 마세요.
- 점수는 브라우저 localStorage에도 저장돼, 오프라인이거나 시트 연결이 잠깐 끊겨도 유실되지 않고 나중에 재전송됩니다.
